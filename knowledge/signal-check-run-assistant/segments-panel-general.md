# General Segment and Panel References

Use these as public-safe starter references. They are not personas and do not represent real users.

## Perspective types

### Fitting

A segment with a plausible problem, reachable context, and clear reason to care.

Use to test whether the idea is understandable and potentially relevant.

### Adjacent

A nearby segment with a related problem but different workflow, budget, trust needs, or urgency.

Use to detect whether the idea is too narrow or too broad.

### Skeptical

A segment with plausible need but strong reasons to distrust, delay, reject, or use alternatives.

Use to surface objections and trust barriers.

### Poor fit

A segment that looks tempting but is unlikely to adopt because the problem, context, channel, or motivation does not fit.

Use to prevent overgeneralization.

### Non-market control

A deliberately irrelevant or unsuitable perspective that should not react positively.

Use to detect over-positive response collapse.

## Starter segment patterns

### Consumer utility

- Fitting: someone with a recurring small annoyance and low tolerance for setup
- Adjacent: someone with the problem but an existing workaround
- Skeptical: someone worried about privacy, lock-in, or maintenance
- Poor fit: someone who rarely experiences the problem
- Non-market control: someone outside the context entirely

### Developer microtool

- Fitting: solo developer with repeated manual cleanup or review task
- Adjacent: team developer using a larger platform
- Skeptical: developer who distrusts generated output or hidden state
- Poor fit: non-technical operator without the workflow
- Non-market control: someone who does not create or maintain code

### Local service

- Fitting: small operator with repetitive scheduling, follow-up, or coordination work
- Adjacent: operator with an existing booking system
- Skeptical: operator relying on personal trust and phone-based workflows
- Poor fit: high-volume business with custom software
- Non-market control: person with no service operation context

## Usage rule

Always rewrite starter segments into the product context. Do not treat these as fixed personas.
