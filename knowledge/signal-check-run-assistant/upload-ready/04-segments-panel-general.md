# Segments and Panel Guidance

Signal Check uses perspectives, not fake personas.

## Required perspective types

- strong_fit: likely to understand the problem
- adjacent_fit: near the problem, but not exact
- skeptical_fit: plausible user with clear objections
- poor_fit: unlikely to care
- non_market_control: not a market target and should not react strongly

## Segment authoring checklist

Each segment should include:

- context
- job or problem
- current alternative
- trust needs
- likely objection
- what would make the idea irrelevant
- reachability for a real test

## Do not include

- invented names
- demographic stereotypes
- fake user quotes
- Do not include claimed purchase intent
- representative-sounding sample language

## Panel balance

A useful panel should make rejection possible. If every perspective sounds positive, the run probably failed.
