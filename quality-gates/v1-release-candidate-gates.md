# v1 Release Candidate Gates

## Gate 1 — Structure

All required v1 artifact folders exist.

## Gate 2 — Parseability

All JSON and YAML files parse.

## Gate 3 — Claim safety

Risky phrases are absent or appear only in explicit anti-claim, limitation, or bad-example contexts.

## Gate 4 — Public example safety

Examples are compact, sanitized, and non-proprietary.

## Gate 5 — Real anchor requirement

No run can be marked complete without a Smallest Real Anchor.

## Gate 6 — Maintainer decision

Final release decision is recorded as:

```text
ready_for_v1
revise_before_v1
block_v1
```
