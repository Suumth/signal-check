# Dogfooding Checklist

Before the run:

- [ ] Use a public-safe idea
- [ ] Avoid private data
- [ ] Start from a clean prompt
- [ ] Keep answers short
- [ ] Capture transcript notes

During the run:

- [ ] One focused question at a time
- [ ] Phase summaries captured
- [ ] Stop conditions observed
- [ ] Negative controls created
- [ ] Claim safety reviewed

After the run:

- [ ] Run-pack files produced
- [ ] YAML files parse
- [ ] Smallest Real Anchor is real
- [ ] Friction log completed
- [ ] Decision recorded
