# Preflight Operator Checklist

## Before the run

- [ ] Product brief is neutral, not pitch-like.
- [ ] Target segment is specific enough to be wrong.
- [ ] At least one skeptical segment is included.
- [ ] At least one poor-fit or non-market control is included.
- [ ] Known assumptions are listed.
- [ ] Claims that must not be made are listed.
- [ ] Kill criteria are drafted before interpretation.

## During the run

- [ ] Reactions are allowed to reject the idea.
- [ ] Confusion and indifference are preserved.
- [ ] Synthetic reactions are not written as real user quotes.
- [ ] Negative controls are interpreted explicitly.
- [ ] Prompt wording is checked for persuasion.
- [ ] Segment collapse is checked.

## Synthesis

- [ ] Includes what was understood.
- [ ] Includes what remained unclear.
- [ ] Includes what was rejected.
- [ ] Includes objections.
- [ ] Includes differences by segment.
- [ ] Includes negative-control interpretation.
- [ ] Includes uncertainty and limits.
- [ ] Does not claim validation.

## Decision

- [ ] Decision is one of continue, revise, narrow, clarify, pause, or kill.
- [ ] Decision is tied to run evidence and counter-evidence.
- [ ] Decision does not jump directly to build.
- [ ] Smallest Real Anchor is specific.
- [ ] Kill or revise threshold is explicit.
