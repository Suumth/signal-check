# Changelog

## Signal Check v1.0-rc.1 — Stable Method Kit Release Candidate

### Added

- Stable v1 method contracts for core artifacts.
- Release-candidate scope document.
- Public README finalization guide.
- v1 repository map.
- v1.0 readiness checklist.
- Release-candidate review template.
- v1 schema set.
- v1 prompt set for final review and release readiness.
- Example acceptance policy for public, sanitized examples.
- Method boundary statement for responsible use.

### Changed

- Consolidates v0.2–v0.9 direction into a stable release-candidate layer.
- Clarifies that Signal Check is a preflight method kit, not a measurement system.
- Makes Smallest Real Anchor mandatory in v1 runs.
- Makes negative controls mandatory in v1 runs.
- Uses verdict language for artifact quality: `ready`, `revise`, `block`.

### Not included

- No CLI.
- No API.
- No web app.
- No model integration.
- No SearCue raw data.
- No private examples.
- No synthetic testimonials.
- No representative claims.
- No validated demand claims.
