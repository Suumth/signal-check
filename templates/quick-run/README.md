# Signal Check Quick Run Template
Use this folder for lightweight Signal Check runs.
A Quick Run is useful when the decision matters, but a full 12-file run would be too heavy.
Copy this folder into a private run location, for example:
```text
../_local-runs/signal-check/<run-id>/
```

or, for temporary local work inside a repository clone:

```text
_local-runs/<run-id>/
```

Do not commit private runs.

For public-safe teaching examples, curate the result separately under:

```text
examples/<example-id>/
```

## Files

| File | Purpose |
| --- | --- |
| `product-brief.md` | Frame the idea, question, audience assumption, constraints, and claim boundary. |
| `test-material.yaml` | Combine segments, stimuli, negative controls, and prompt-robustness notes. |
| `reaction-notes.md` | Capture short raw reactions and counter-patterns before interpretation. |
| `preflight-report.md` | Summarize bounded findings, uncertainty, and claims not allowed. |
| `smallest-real-anchor.md` | Define the next real-world test. |

## Boundary

A Quick Run is not market research and does not validate demand, purchase intent, conversion, willingness to pay, or product-market fit.
