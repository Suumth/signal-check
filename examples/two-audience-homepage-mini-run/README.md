# Two-Audience Homepage Mini Run
This is a public-safe teaching example for Signal Check.
It uses a fictional product-lab homepage problem:
A homepage must explain both:
1. a concrete proof product, and
2. the broader product method behind it.
The example demonstrates how Signal Check can help compare homepage story structures without treating synthetic reactions as real user evidence.
This example is not market research, not validation, and not proof that any product should be built.
