# Kill Criteria

Kill criteria are written before interpretation.

Good kill criteria are concrete enough to change the next action. They do not need to kill the entire product. They may kill a segment, positioning angle, feature bundle, channel, or pricing assumption.

Examples:

- If fitting segments cannot explain the value in their own words, revise the category before building.
- If skeptical segments raise trust issues that the product cannot answer, do not build the current version.
- If adjacent segments consistently prefer an existing workaround, test a narrower wedge.
- If poor-fit segments react positively for vague reasons, treat that as noise, not signal.

Every preflight report should include keep, change, and stop decisions.
