# Scoring

Signal Check uses scoring only as structured classification against anchors.

The purpose is not to validate an idea. The purpose is to make interpretation explicit enough that another person can see the evidence, caveat, and uncertainty behind each judgment.

Recommended dimensions:

- comprehension
- problem recognition
- urgency
- current workaround
- trust friction
- switching friction
- differentiation
- real-world next step

Each dimension should include:

- anchor label
- evidence excerpt or observation
- uncertainty note
- counter-evidence
- reviewer note

Avoid single-number summaries. If a summary is needed, use language such as weak, mixed, directional, or worth a real anchor. Never present a score as demand, purchase likelihood, or validation.
