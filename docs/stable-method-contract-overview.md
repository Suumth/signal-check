# Stable Method Contract Overview

v1 introduces stable contracts for the core Signal Check artifacts.

## Contract list

| Contract | Purpose |
|---|---|
| Product Brief | Define the idea without overselling |
| Segment | Describe a perspective without fake persona detail |
| Panel Composition | Combine fit, adjacent, skeptical, poor-fit, and control perspectives |
| Negative Controls | Detect over-positive prompts or weak differentiation |
| Stimuli | Define messages or concepts to test |
| Raw Reactions | Preserve reactions before synthesis |
| Synthesis | Summarize without cherry-picking or validation language |
| Bias Audit | Inspect prompt and interpretation risks |
| Smallest Real Anchor | Translate output into the next real test |
| Quality Gates | Decide ready, revise, or block |

## Stability expectations

A v1 artifact should be:
- small enough to review manually
- explicit about uncertainty
- traceable to its inputs
- claim-safe
- usable without a model-specific dependency
