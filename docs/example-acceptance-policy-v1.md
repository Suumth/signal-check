# Example Acceptance Policy — v1

Public examples are teaching artifacts, not proof of product quality.

## Acceptable examples

A public example may be accepted if it is:
- sanitized
- compact
- non-proprietary
- free of private beta data
- explicit about limitations
- structured around learning value
- clear about the Smallest Real Anchor

## Unacceptable examples

Reject examples that include:
- private customer data
- internal product secrets
- large raw response dumps
- synthetic testimonials
- claims of validated demand
- claims of representative evidence
- named real people without permission
- sensitive personal data

## Example maturity levels

| Level | Meaning |
|---|---|
| `sketch` | illustrates a template only |
| `teaching` | suitable for public learning |
| `reference` | high-quality example for reuse |
| `blocked` | included only as a bad-example teaching case |

## Required note

Every public example should include:

```text
This example is a teaching artifact. It does not show real market evidence or validated demand.
```
