# v1.0-rc.1 Scope — Stable Method Kit Release Candidate

## Release intent

v1.0-rc.1 is a release candidate for Signal Check as a stable open-source method kit.

The release candidate should be evaluated against this question:

> Can a careful builder use Signal Check without confusing synthetic perspective work for real user evidence?

## In scope

- stable method contracts
- public documentation polish
- release-candidate review
- v1 schemas and templates
- v1 run acceptance gates
- example acceptance policy
- maintainer release review
- strict claim boundaries

## Out of scope

- command-line interface
- hosted workflow
- model integration
- survey tooling
- analytics
- real-user research platform
- market sizing
- purchase-intent measurement
- private product data

## Required v1 properties

A v1-compatible Signal Check run must include:

1. Product brief.
2. Segment plan.
3. At least one skeptical perspective.
4. At least one poor-fit perspective.
5. At least one negative control.
6. Stimulus set.
7. Raw reactions or reaction notes.
8. Synthesis with uncertainty preserved.
9. Bias audit.
10. Smallest Real Anchor.

## Release-candidate pass condition

The RC passes only if:
- examples are public-safe
- claims are conservative
- artifact contracts are coherent
- a new user can follow the manual workflow
- failure modes are visible
- no synthetic output is framed as validated market evidence
