# Signal Check v1.0.0-rc.2

Signal Check is a lightweight open-source method kit for testing product ideas before you build.

This release candidate includes the core file-based workflow for running a structured product preflight check with LLM assistance.

## Changes since rc.1

- Polished the public README and public method-kit language
- Updated public documentation and changelog
- Added public-safe GitHub issue templates
- Added the data-minimal `quiet-invoice-mini-run` example
- Kept the project file-based: no CLI, API, web app, package setup, or model integration

## Included

- product brief template
- segment planning template
- stimuli template
- negative controls
- prompt robustness plan
- bias audit prompt
- scoring and summary structure
- real validation planning template
- example run structure
- quality gates

## What this release is for

Use this release to test whether Signal Check is understandable, useful, and practical for early-stage product decisions.

It is especially suited for:

- solo builders
- indie hackers
- AI-assisted developers
- small product teams
- open-source maintainers
- product marketers

## What this release is not

This is not a market research platform, statistical study tool, survey system, or proof of demand.

Signal Check helps structure early judgment.
It does not replace real user research.

## Recommended use

Run Signal Check before:

- building an MVP
- writing a coding-agent prompt
- launching a landing page
- committing to a feature
- making strong product claims

## Feedback wanted

Feedback is especially useful on:

- clarity of the README
- ease of running the templates
- usefulness of the example run
- whether the quality gates prevent overclaiming
- where the workflow feels too heavy or too manual

## Safety note

Signal Check does not tell you whether an idea will succeed.
It helps you decide what to test next before you build too much.
