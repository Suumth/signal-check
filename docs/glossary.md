# Glossary

## Signal Check

The project: a file-based open-source method kit for checking product ideas before building them.

## Product Preflight

The manual workflow used to frame an idea, select segments, collect free reactions, classify against anchors, audit bias, apply kill criteria, and choose a Smallest Real Anchor.

## Synthetic Panel Kit

The method layer made of segment files, prompts, templates, controls, schemas, and example artifacts. It helps structure thinking but does not replace real research.

## Segment

A bounded audience context with a situation, problem or job, constraints, current alternatives, likely objections, trust needs, reachability, and exclusion rules.

## Persona

A fictional character profile. Signal Check avoids persona-first work because it often creates false specificity.

## Stimulus

The artifact shown for reaction: a brief, landing-page copy, concept, flow, or feature description.

## Negative Control

A deliberately weak, irrelevant, or misleading stimulus used to test whether the process is over-reading positive reactions.

## Non-Market Control

A deliberately poor-fit or irrelevant segment used to check whether the process converts non-market reactions into false support.

## Prompt Robustness

The practice of comparing neutral prompt variants before treating a pattern as stable.

## Bias Audit

A review for leading prompts, over-positivity, negative-control failure, prompt sensitivity, cherry-picking, response collapse, and missing counter-evidence.

## Kill Criteria

Pre-written conditions that change or stop the next action.

## Smallest Real Anchor

The smallest real-world action that can reduce uncertainty after the preflight.

## Reproducibility Contract

The expectation that every run preserves enough inputs, raw reactions, controls, interpretations, and edits to be reviewed later.
