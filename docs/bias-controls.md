# Bias Controls

Bias controls protect the preflight from becoming a story the maker wants to hear.

Required controls:

- Use neutral prompts.
- Ask for free reactions before structured classification.
- Include skeptical and poor-fit segments.
- Include a non-market control when an irrelevant segment could expose false support.
- Include negative controls.
- Compare prompt variants.
- Keep raw responses.
- Record counter-evidence.
- Separate evidence from interpretation.
- Write kill criteria before reviewing reactions.

Common bias risks:

- leading language
- over-specific founder framing
- cherry-picked positive snippets
- over-positivity
- negative-control failure
- prompt sensitivity
- response collapse
- treating fluency as truth
- treating synthetic reactions as people
- collapsing uncertainty into a neat conclusion

## Failure Signals

Treat the run as low confidence when:

- broad generic controls receive strong reactions
- poor-fit or non-market-control segments appear supportive for vague reasons
- all segments sound similar
- a finding disappears under neutral wording
- the summary has no material counter-evidence

The right response is usually to narrow the claim, rerun a prompt variant, revise the stimulus, or move to a smaller real-world anchor.
