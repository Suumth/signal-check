# Public README Finalization Guide

The v1 README should make the project understandable in under two minutes.

## Required structure

1. What Signal Check is.
2. What it is not.
3. Who it helps.
4. How a manual run works.
5. What files are used.
6. What outputs mean.
7. How to avoid fake validation.
8. How to contribute.

## Opening positioning

Recommended opening:

> Signal Check is an open-source method kit for checking product ideas before building. It helps teams and solo builders structure target segments, messages, objections, risks, and the next real test. It does not simulate validated demand or replace talking to real users.

## Must include

- segment-first, not persona-first
- skeptical and poor-fit perspectives
- negative controls
- explicit refusal allowed
- Smallest Real Anchor
- no representative claims
- no synthetic testimonials
- no validated purchase-intent claims

## Avoid

Avoid language such as:

```text
- AI market research
- proves your idea
- validates demand
- predicts customers
- representative panel
- avoid synthetic users said
- avoid guaranteed product-market fit
```

## Better wording

Use:

```text
- early product preflight
- structured critique
- assumption exposure
- directional synthesis
- next real test
- claim-safe artifact
- method kit
```
