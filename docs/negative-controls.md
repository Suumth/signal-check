# Negative-Control Design

## Purpose

Negative controls help detect leading prompts, overbroad value propositions, and synthetic positivity.

A negative control is not a hostile reviewer. It is a deliberate mismatch that should create resistance.

## Types of negative controls

### Non-market control

A perspective with no meaningful reason to adopt the product.

### Overclaim control

A stimulus that makes a claim the project must not make. The expected reaction should reject or flag the claim.

### Hype-language control

A stimulus with inflated language. The expected reaction should reveal distrust, vagueness, or overpromising.

### Channel mismatch control

A perspective that is unlikely to be reachable through the proposed acquisition channel.

### Price-sensitivity control

A perspective that tests whether perceived value collapses under a realistic cost or switching burden.

## Pass/fail guidance

A negative control passes when it resists, flags, or rejects the relevant stimulus.

A negative control fails when it becomes enthusiastic without a credible reason.

## Required documentation

For each negative control, record:

- What it is controlling for
- Expected resistance
- What failure would imply
- How the next prompt or stimulus should change if it fails
