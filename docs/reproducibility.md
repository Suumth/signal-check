# Reproducibility

A useful Product Preflight can be reviewed later.

The goal is traceability, not bit-identical repetition. Model behavior, prompt wording, human edits, and context can change. Signal Check therefore preserves the materials needed to understand how a conclusion was reached.

Keep:

- product brief
- selected segment files
- sampling plan
- prompt versions
- stimuli
- negative controls
- non-market-control notes
- prompt-robustness plan
- raw responses
- scored or classified results
- bias audit
- kill criteria result
- preflight report
- Smallest Real Anchor

Do not overwrite raw material. If you rerun a prompt, save it as a new run.

Document model, date, prompt version, and manual edits when relevant. If a human reviewer changes an interpretation, record the reason.

## Reproducibility Contract

A run should make it possible to answer:

- What was the original product assumption?
- Which segments were used and why?
- Which controls were used?
- Which prompts and variants were used?
- What raw reactions were preserved?
- What evidence and counter-evidence supported the summary?
- What uncertainty remained?
- What real-world anchor was selected?

Do not edit old outputs to make the run cleaner. Add a new note or a new run instead.
