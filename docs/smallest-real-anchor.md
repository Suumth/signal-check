# Real-Test Translation

## Purpose

The final step of a panel-kit run is not a score. It is a real next test.

Signal Check calls this the Smallest Real Anchor.

## Translation process

1. Identify the riskiest remaining assumption.
2. Convert it into observable behavior.
3. Define the smallest audience or channel where it can be tested.
4. Define what counts as signal.
5. Define what counts as non-signal.
6. Define kill or revise criteria.
7. Define ethical and claim boundaries.

## Good Smallest Real Anchors

A good anchor is:

- Real
- Small
- Observable
- Cheap enough to run soon
- Linked to a specific assumption
- Clear about what would stop or change the idea

## Bad Smallest Real Anchors

A bad anchor is:

- Another synthetic prompt
- A vague landing page with no success criteria
- A vanity poll
- A broad “ask people if they like it” exercise
- A test that cannot falsify the assumption

## Output format

Each translation should specify:

- Assumption
- Test format
- Audience/channel
- Stimulus
- Signal threshold
- Non-signal threshold
- Kill criteria
- Next decision
