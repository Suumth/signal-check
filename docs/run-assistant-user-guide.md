# Run Assistant User Guide

This guide explains how to use the optional Signal Check Run Assistant.

## Start a new run

Begin with a short idea description. The assistant should ask focused setup questions before generating files.

## Work through four phases

1. Setup
2. Test Material
3. Analysis
4. Output

## Expect pauses

The assistant should pause when:

- the brief is too vague
- negative controls are missing
- a negative control succeeds
- synthesis becomes overconfident
- no real-world anchor is available

## Final output

At the end, request either:

- a ZIP containing the 12 run-pack files, or
- individually named code blocks if file generation is unavailable

## Review before use

Before acting on the output, review claim safety and the Smallest Real Anchor.
