# Knowledge File Update Policy

Knowledge files exist to make the optional Custom GPT easier to configure.

## Allowed

- concise method summaries
- lite templates
- public-safe examples
- phase maps
- prompt-to-file mapping
- guardrails

## Not allowed

- private product data
- raw interviews
- internal strategy
- customer data
- proprietary screenshots
- large raw responses
- content that claims validation

## Update process

1. Change the source repo file.
2. Update the relevant knowledge file.
3. Run quality gates.
4. Run assistant smoke test.
5. Record the result.
