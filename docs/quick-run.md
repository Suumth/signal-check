# Quick Run
A Quick Run is a lightweight Signal Check mode for smaller product, message, or positioning decisions.
It is still a preflight method.
It is not validation, market research, representative evidence, or real user feedback.
## Use Quick Run for
- homepage hero direction
- product section order
- beta invite wording
- feature naming
- README positioning
- lightweight message comparisons
- early "does this sound clear enough to test?" questions
## Do not use Quick Run for
- pricing validation
- willingness-to-pay claims
- market sizing
- demand claims
- safety or compliance claims
- launch-readiness decisions
- high-stakes product commitments
- decisions where real behavior is required
## Quick Run output set
A Quick Run should produce five files:
```text
product-brief.md
test-material.yaml
reaction-notes.md
preflight-report.md
smallest-real-anchor.md
```

## What can be compressed

A Quick Run may combine:

* sampling plan
* selected segments
* stimuli
* negative controls
* prompt robustness notes

into:

```text
test-material.yaml
```

A Quick Run may combine:

* scored results
* bias audit
* claim-safety notes
* segment differences

into:

```text
preflight-report.md
```

## What must not be skipped

A Quick Run must still include:

* a clear run question
* at least one fitting perspective
* one skeptical or poor-fit perspective
* at least one negative control
* a claim boundary
* counter-evidence
* uncertainty
* a Smallest Real Anchor

## Decision rule

A Quick Run should end with one of:

* Chat is enough
* Reframe
* Narrow
* Full Run needed
* Real-world test needed
* Build only a small reversible artifact

It should not end with:

* validated
* proven
* users want this
* market confirmed
* build the full product now
