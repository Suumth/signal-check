# Run Assistant Release Process

Use this when publishing updates that include assistant support files.

## Steps

1. Confirm core method files are stable.
2. Update assistant docs or knowledge files.
3. Run repository quality gates.
4. Run assistant smoke test.
5. Record smoke-test result.
6. Commit with a clear message.
7. Do not tag unless the repository release process calls for it.

## Suggested commit messages

- `Add Signal Check run assistant builder docs`
- `Add Signal Check run assistant dogfooding kit`
- `Update Signal Check run assistant knowledge files`
- `Polish Signal Check run assistant instructions`

## Release note language

Always describe the assistant as optional.
Never describe it as validation, market research, or real user simulation.
