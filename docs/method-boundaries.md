# Method Boundaries — v1

Signal Check is a preflight method, not a measurement instrument.

## It can help with

- clarifying a product idea
- separating segments from personas
- identifying weak messages
- surfacing objections
- testing prompt sensitivity
- designing negative controls
- writing a next real test
- documenting assumptions carefully

## It cannot establish

- market demand
- willingness to pay
- representative user opinion
- statistically meaningful preference
- real conversion likelihood
- safety, compliance, or legal sufficiency
- product-market fit

## Boundary language

Use this sentence in public materials:

> Signal Check produces structured, synthetic critique for earlier thinking. It is not evidence that real users want, need, trust, or will buy the product.

## Evidence ladder

Signal Check output sits below:
- interviews
- waitlist behavior
- landing page experiments
- prototype usage
- paid pilots
- real retention data

It should point toward those real anchors rather than pretending to replace them.
