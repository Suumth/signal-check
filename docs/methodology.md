# Methodology

Signal Check v0.1 uses a manual Product Preflight workflow. The method has eight phases.

## 1. Frame The Idea

Write a brief that names the product idea, the primary segment assumption, the problem, the proposed behavior change, current alternatives, trust needs, reachability, the riskiest assumption, kill criteria, and a candidate Smallest Real Anchor.

## 2. Select Segments

Choose segments before prompts. Avoid invented personalities. A useful segment describes context, problem or job, constraints, current workarounds, likely objections, trust needs, reachability, exclusions, and assumptions not to make.

Every preflight should include:

- fitting perspectives
- adjacent perspectives
- skeptical perspectives
- poor-fit perspectives
- at least one non-market control when false support is a risk

## 3. Prepare Stimuli

Stimuli are short descriptions, landing-page copy, onboarding flows, concept flows, compatibility notes, or feature summaries. Keep them plain enough that reactions target the idea, not polish.

Each stimulus should include claim boundaries. Do not imply partnerships, official support, safety guarantees, representative evidence, or measured purchase intent unless those facts exist outside the kit.

## 4. Elicit Free Reactions

Ask for open reactions before any structured classification. Look for confusion, category mismatch, urgency, alternatives, trust concerns, and missing context.

Do not ask for a 1-5 score. Do not ask whether the respondent would buy. Do not turn reactions into testimonials.

## 5. Check Prompt Robustness

Compare neutral prompt variants before trusting a pattern. Treat a finding as unstable when it depends on wording, disappears without founder-friendly context, or is stronger in a negative control.

## 6. Classify Against Anchors

Use scoring dimensions as anchors for structured interpretation. Each classification needs evidence, uncertainty, and a caveat.

This is structured classification, not validation. Do not collapse classifications into a single product score or grade.

## 7. Audit Bias And Apply Kill Criteria

Run a bias audit before summary. Check over-positivity, negative-control failure, prompt sensitivity, cherry-picking, response collapse, missing counter-evidence, and segment bias.

Apply kill criteria before deciding what to keep, change, or stop.

## 8. Decide The Next Real Anchor

A Product Preflight ends only when it names the smallest real-world action that could reduce uncertainty. The action must leave the prompt environment: real conversations, beta replies, landing-page tests, concierge tests, observation of current workarounds, or similar real anchors.
