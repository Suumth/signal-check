# Methodology

The method has six phases.

## 1. Frame The Idea

Write a brief that names the product idea, the assumed user segment, the problem, the proposed behavior change, and the riskiest assumption.

## 2. Select Segments

Choose segments before prompts. Avoid invented personalities. A useful segment describes context, constraints, buying or adoption posture, and exclusion rules.

Every preflight should include:

- fitting perspectives
- adjacent perspectives
- skeptical perspectives
- poor-fit perspectives

## 3. Prepare Stimuli

Stimuli are short descriptions, landing-page copy, onboarding flows, mock reviews, or feature summaries. Keep them plain enough that reactions target the idea, not polish.

## 4. Elicit Free Reactions

Ask for open reactions before any structured classification. Look for confusion, category mismatch, urgency, alternatives, trust concerns, and missing context.

## 5. Classify Against Anchors

Use scoring dimensions as anchors for structured interpretation. Each classification needs evidence, uncertainty, and a caveat.

## 6. Decide The Next Real Anchor

A preflight ends only when it names the smallest real-world action that could reduce uncertainty.
